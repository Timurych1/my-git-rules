## B2. Удаленное изменение – это push
#### `git push <remote> <local_branch>:<remote_branch>` – добавить изменения из локальной ветки `<local_branch>` и переместить соответствующую ветку удаленного репозитория
#### `git push <remote>` – выполнить `push` для всех веток, у которых ветка на удаленном репозитории не сдвигалась (использовать вместо `git push <remote> <branch>`)
#### `git push --tags` – отправить все теги в удаленный репозиторий
#### `git push --mirror` – выполнить агрессивный `push` для всех веток, тегов и HEAD, подходит для создания удаленной копии локального репозитория
#### `git push --force-with-lease` – выполнить push с проверкой, что удаленная ветка не изменилась
