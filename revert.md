## A4. Revert — безопасная отмена коммитов

- `git revert <commit>` - создать новый коммит, отменяющий изменения из <commit>
- `git revert HEAD` - отменить последний коммит
- `git revert HEAD~1` - отменить предпоследний коммит
- `git revert <commit1>..<commit2>` - отменить диапазон коммитов
- `git revert --no-commit <commit>` - подготовить отмену, но не создавать коммит (можно просмотреть изменения)
- `git revert --abort` - отменить正在进行 revert (если возник конфликт)

### Revert vs Reset:
- **Reset** — удаляет коммиты из истории (опасно для публичных веток!)
- **Revert** — создаёт новый коммит, сохраняя историю (безопасно!)
